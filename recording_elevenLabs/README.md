# FIFA 2026 UAT Chat — Narrated Demo (ElevenLabs voice)

Same demo as `../recordings/`, but the injected narration is generated with **ElevenLabs**
cloud TTS instead of macOS `say`.

## What the demo shows
Running on a **third-party custom website that simulates the FIFA 2026 site** (an external site to
the Salesforce org), it demonstrates how to **add a custom Quit button that ends the Iris agent chat
without changing the out-of-the-box MIAW component**. The `Quit` button is a host-page control
(`#amaQuit`) that calls `userVerificationAPI.clearSession()` + `utilAPI.removeAllComponents()` and
navigates back to the home page.

## Deliverables (two voice variants)
- **`fifa26-uat-chat-narrated.mp4`** — ~6.5 MB, ~115s — voice **Sarah** (`EXAVITQu4vr4xnSDxMaL`, clear female)
- **`fifa26-uat-chat-narrated-chris.mp4`** — ~5.9 MB, ~107s — voice **Chris** (`iP95p4xoKVk53GoZ742B`, casual male)

Both are H.264 video + AAC narration, model `eleven_multilingual_v2`. Timelines: `timeline.json` (Sarah), `timeline-chris.json` (Chris). Regenerate either with `ELEVEN_VOICE_ID=<id> node record_eleven.js` + `mux_eleven.py`.

## Flow recorded
1. Home page → pass auth gate (`sfdc`/`admin123`) in ~2s → select USA region
2. General-idea intro narrated **on the home page**
3. Visible cursor + **“Chat with Iris (UAT)”** callout hovers the widget (>=2s) → click → full-page mode
4. Pre-chat form (name `Jin`/`Wang` + email `eternalwangjs@gmail.com`) → Start Conversation
5. Iris welcome → ask **“schedule for today”** → agent reply
6. **“thanks”** → agent sends a short feedback survey (scrolled into view)
7. Click the custom **Quit** button (ripple + pulse effect) → UI switches back to the home page
8. Click the Iris (UAT) widget again → a brand-new conversation starts (fresh pre-chat form)
9. Closing statement

## Pipeline (ElevenLabs)
```
record_eleven.js   # Playwright flow + records video; pre-generates clips via ElevenLabs to
                   #   measure duration and drive audio-led holds; reuses those clips for the mux
   └─ eleven_tts.py   # one narration clip via ElevenLabs REST API -> 48k WAV (per beat)
mux_eleven.py      # anchor clips = max(event, prev_end) -> adelay + amix -> freeze-pad -> encode MP4
```

## Reproduce
```bash
export ELEVENLABS_API_KEY="sk_..."          # required (cloud API)
export ELEVEN_VOICE_ID="EXAVITQu4vr4xnSDxMaL"  # Sarah (female); or George=JBFqnCBsd6RMkjVDRZzb
cd /tmp/pw-eleven                            # needs playwright + chromium
node record_eleven.js                        # -> timeline.json + clips/*.wav + video/*.webm
python3 mux_eleven.py "<repo>/recording_elevenLabs/fifa26-uat-chat-narrated.mp4"
```

## Notes
- **Free-plan constraint:** ElevenLabs free accounts cannot use *library* voices (e.g. Rachel) via
  the API — only the default premade voices. Verified working on free: **Sarah**, **Laura** (female),
  **George** (male). Rachel/Adam/Aria returned HTTP 402 (paid plan required).
- The API key is read from the `ELEVENLABS_API_KEY` environment variable only — never hard-coded.
- Cost: ~14 short clips (~1,400 characters) per full render.
- Architecture/flow diagrams: see `../recordings/architecture.html` (the pipeline is identical;
  only the TTS engine differs — `say` → ElevenLabs `eleven_tts.py`).
