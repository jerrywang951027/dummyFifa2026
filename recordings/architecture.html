Narrated Browser-Demo Recorder
Architecture & Flow

How a scripted browser walkthrough of the Iris (UAT) chat is turned into an MP4 with synced injected narration — combining installed agent skills (patterns) with local, offline tooling (execution), glued by three custom scripts using an audio-led timing model.

1 Component Architecture

Four layers. Agent skills supply the patterns; local applications do the work; custom scripts are the glue; SF Pi tooling supports the loop.

Agent skills (patterns) Local apps / CLIs (execution) Custom scripts (glue) SF Pi tooling (support)

Agent skills are guidance only — they shaped the architecture but execute nothing. All actual work is done by local, offline tools driven by the custom scripts.

2 End-to-End Pipeline Flowchart

From the requirement (captured in a prior pi session) to the final MP4. Note the two feedback loops that verify quality.

3 The Sync Model — “Audio-led” Timing

The core trick that keeps narration matched to on-screen actions: the video waits for the words, not the other way around.

Clips are anchored at max(event_time, prev_clip_end) so they never overlap; the tail is freeze-padded so the closing line always finishes on screen.

Problem observedFix in the model
Audio ran ahead of the UIHold each opening screen for its clip duration
Reached full-page before narration doneSpeak the “click” line fully, then navigate
Blank pause on full-page loadMove prechat line to play during load
Survey not visibleScroll Submit Feedback into view
Cursor/tooltip not recordedInject visible cursor + callout + pulse
Closing line cut offFreeze-pad video tail to fit audio
